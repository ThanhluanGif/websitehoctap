# GIÁO ÁN HỌC PHẦN: XÂY DỰNG WEBSITE VỚI HTML, CSS VÀ GITHUB

## 1. Thông tin chung

| Hạng mục | Nội dung |
|---|---|
| Tên học phần | Xây dựng giao diện website với HTML, CSS và GitHub |
| Thời lượng | 12 buổi, triển khai trong 3 tháng |
| Tần suất | 1 buổi/tuần |
| Thời lượng gợi ý | 2-3 giờ/buổi |
| Hình thức học | Học theo dự án, kết hợp hướng dẫn trực tiếp và thực hành trên máy |
| Sản phẩm cuối khóa | Website cá nhân hoặc landing page hoàn chỉnh, responsive, có mã nguồn trên GitHub và được triển khai bằng GitHub Pages |

## 2. Mô tả học phần

Học phần này dành cho người mới bắt đầu học lập trình web. Trong 12 buổi, học viên đi từ cách tạo một trang HTML đầu tiên đến xây dựng một website hoàn chỉnh bằng HTML, CSS và Git/GitHub.

Nội dung được tổ chức theo hướng thực hành dự án. Ngay từ buổi đầu, học viên bắt đầu xây dựng một website cá nhân hoặc landing page đơn giản. Mỗi buổi học bổ sung thêm một phần kỹ năng mới: cấu trúc HTML, định dạng CSS, bố cục bằng Flexbox/Grid, responsive design, hiệu ứng giao diện, quản lý mã nguồn bằng Git và triển khai website lên GitHub Pages.

Cuối khóa, học viên có một sản phẩm có thể mở bằng trình duyệt, chia sẻ qua đường link và trình bày trước lớp. Mục tiêu không chỉ là viết đúng cú pháp, mà còn hình thành thói quen làm việc rõ ràng: đặt tên file hợp lý, tổ chức thư mục, kiểm tra giao diện trên nhiều màn hình, commit code có ý nghĩa và trình bày sản phẩm mạch lạc.

## 3. Đối tượng học viên

- Học viên chưa từng học lập trình web hoặc mới tiếp xúc ở mức cơ bản.
- Học sinh, sinh viên hoặc người đi làm muốn tự xây dựng website cá nhân, trang giới thiệu sản phẩm, trang sự kiện hoặc portfolio.
- Người cần nền tảng HTML, CSS và GitHub trước khi học JavaScript, React hoặc các framework frontend khác.

## 4. Yêu cầu đầu vào

Học viên cần đáp ứng các điều kiện tối thiểu sau:

- Sử dụng máy tính ở mức cơ bản: tạo thư mục, lưu file, cài đặt phần mềm.
- Biết dùng trình duyệt web và tìm kiếm thông tin trên Internet.
- Có máy tính cá nhân để thực hành trên lớp và làm bài tập ở nhà.
- Có email cá nhân để đăng ký tài khoản GitHub.
- Không yêu cầu kinh nghiệm lập trình trước đó.

## 5. Chuẩn đầu ra

Sau khi hoàn thành học phần, học viên có thể:

- Tạo được cấu trúc cơ bản của một website gồm HTML, CSS, hình ảnh và tài liệu README.
- Viết HTML rõ ràng, đúng vai trò nội dung và sử dụng được các thẻ semantic phổ biến.
- Dùng CSS để định dạng màu sắc, chữ, khoảng cách, border, card, button và các thành phần giao diện cơ bản.
- Xây dựng layout bằng Flexbox và CSS Grid trong các tình huống thực tế như navbar, hero section, danh sách card, gallery và bố cục trang.
- Điều chỉnh website hiển thị tốt trên desktop, tablet và mobile bằng media query.
- Thêm được hiệu ứng hover, transition và animation nhẹ ở mức phù hợp.
- Sử dụng Git để theo dõi thay đổi của project và đưa mã nguồn lên GitHub.
- Triển khai website tĩnh bằng GitHub Pages.
- Trình bày được ý tưởng, cấu trúc và quá trình hoàn thiện sản phẩm cuối khóa.

## 6. Phương pháp giảng dạy

- **Học theo dự án:** Mỗi buổi đóng góp trực tiếp vào website cá nhân hoặc landing page của học viên.
- **Giảng ngắn, làm nhiều:** Giảng viên giới thiệu khái niệm qua ví dụ nhỏ, sau đó học viên thực hành ngay trên project.
- **Code cùng giảng viên:** Các phần khó như layout, responsive và Git được thực hiện theo từng bước để học viên quan sát quy trình làm thật.
- **Sửa lỗi công khai có chọn lọc:** Giảng viên chọn một số lỗi phổ biến trong lớp để phân tích, giúp học viên biết cách đọc lỗi và tự kiểm tra.
- **Review sản phẩm định kỳ:** Sau mỗi nhóm kỹ năng, học viên được nhận góp ý về giao diện, cấu trúc file và cách viết code.
- **Khuyến khích cá nhân hóa:** Học viên được chọn chủ đề website phù hợp với sở thích, nhưng vẫn phải đáp ứng checklist kỹ thuật chung.

## 7. Phương pháp đánh giá

# KẾ HOẠCH CHI TIẾT TỪNG BUỔI

## Buổi 1: Làm quen với Web, HTML cơ bản và môi trường làm việc

### Mục tiêu cụ thể

- Phân biệt được vai trò của trình duyệt, HTML, CSS và JavaScript trong một website.
- Tạo được thư mục project đầu tiên và file `index.html`.
- Viết được một trang giới thiệu bản thân đơn giản bằng các thẻ HTML cơ bản.

### Kiến thức trọng tâm

- Website hoạt động như thế nào ở mức khái quát.
- Vai trò của HTML, CSS và JavaScript.
- Cài đặt và làm quen với Visual Studio Code.
- Cấu trúc file HTML: `<!DOCTYPE html>`, `html`, `head`, `body`.
- Các thẻ thường dùng: `h1` đến `h6`, `p`, `a`, `img`, `br`, `hr`.
- Cách đặt tên file, tên thư mục và lưu tài nguyên ảnh.

### Hoạt động của giảng viên

- Minh họa một trang web hoàn chỉnh, sau đó tách thành các phần HTML, CSS và hình ảnh.
- Hướng dẫn cài đặt VS Code, mở folder project và tạo file đầu tiên.
- Code mẫu một trang giới thiệu ngắn, vừa làm vừa giải thích ý nghĩa từng thẻ.
- Kiểm tra nhanh máy của học viên, xử lý lỗi lưu file, lỗi mở sai đường dẫn hoặc ảnh không hiển thị.

### Hoạt động của học viên

- Tạo folder project cá nhân.
- Viết file `index.html` theo mẫu, sau đó thay nội dung bằng thông tin của mình.
- Thêm ảnh đại diện và ít nhất một liên kết ngoài.
- Mở file bằng trình duyệt và chỉnh sửa lại khi phát hiện lỗi.

### Sản phẩm cần hoàn thành sau buổi học

Một trang HTML giới thiệu bản thân gồm tiêu đề, đoạn mô tả ngắn, ảnh đại diện, danh sách sở thích ở dạng văn bản và một liên kết ngoài.

### Bài tập về nhà

Hoàn thiện trang "Giới thiệu bản thân" với các nội dung:

- Họ tên hoặc biệt danh muốn hiển thị.
- Ảnh đại diện lưu trong thư mục project.
- Một đoạn giới thiệu 5-7 câu.
- Tối thiểu 3 sở thích hoặc mục tiêu học tập.
- Liên kết đến GitHub, Facebook hoặc một website yêu thích.

### Tiêu chí đánh giá ngắn gọn

- File mở được trên trình duyệt.
- HTML có đủ cấu trúc cơ bản.
- Đường dẫn ảnh và liên kết hoạt động.
- Nội dung được cá nhân hóa, không để nguyên dữ liệu mẫu.

## Buổi 2: Tổ chức nội dung với HTML semantic, danh sách, bảng và form

### Mục tiêu cụ thể

- Sắp xếp nội dung website bằng các khu vực rõ ràng như header, main, section và footer.
- Tạo được danh sách, bảng thông tin đơn giản và form nhập liệu cơ bản.
- Nâng cấp trang cá nhân thành một bố cục có cấu trúc thay vì chỉ là các dòng nội dung rời rạc.

### Kiến thức trọng tâm

- Danh sách: `ul`, `ol`, `li`.
- Bảng: `table`, `thead`, `tbody`, `tr`, `th`, `td`.
- Form: `form`, `label`, `input`, `textarea`, `select`, `button`.
- Các loại input phổ biến: text, email, tel, radio, checkbox.
- Semantic HTML: `header`, `nav`, `main`, `section`, `article`, `aside`, `footer`.
- Tầm quan trọng của `label` và cấu trúc nội dung dễ đọc.

### Hoạt động của giảng viên

- So sánh một trang chỉ dùng nhiều thẻ `div` với một trang có semantic HTML rõ ràng.
- Hướng dẫn chia website cá nhân thành các phần: giới thiệu, kỹ năng/sở thích, lịch học hoặc dự án, liên hệ.
- Code mẫu form liên hệ đơn giản và giải thích mối quan hệ giữa `label` và `input`.
- Gợi ý cách chọn nội dung phù hợp với project cuối khóa.

### Hoạt động của học viên

- Chuyển trang buổi 1 sang cấu trúc semantic HTML.
- Tạo danh sách kỹ năng, sở thích hoặc mục tiêu học tập.
- Tạo một bảng nhỏ, ví dụ lịch học, kế hoạch tuần hoặc danh sách dự án dự kiến.
- Thêm form liên hệ hoặc form đăng ký nhận thông tin.

### Sản phẩm cần hoàn thành sau buổi học

Phiên bản HTML có cấu trúc rõ ràng của website cá nhân/landing page, gồm ít nhất 4 khu vực nội dung và một form cơ bản.

### Bài tập về nhà

Hoàn thiện phần nội dung cho website:

- Header có tên website và menu điều hướng.
- Main có ít nhất 3 section.
- Footer có thông tin liên hệ hoặc ghi chú bản quyền.
- Form có tối thiểu 3 trường nhập liệu và nút gửi.
- Nội dung không dùng văn bản giữ chỗ như "Lorem ipsum" ở các phần chính.

### Tiêu chí đánh giá ngắn gọn

- Dùng đúng các thẻ semantic ở vị trí hợp lý.
- Form có `label` rõ ràng.
- Nội dung đủ để tiếp tục thiết kế giao diện ở các buổi sau.
- Code thụt dòng dễ đọc.

## Buổi 3: CSS cơ bản, selector, màu sắc và chữ

### Mục tiêu cụ thể

- Kết nối được file CSS riêng với HTML.
- Dùng selector để định dạng đúng phần tử mong muốn.
- Thiết lập được phong cách thị giác ban đầu cho website: màu nền, màu chữ, font, kích thước và căn chỉnh.

### Kiến thức trọng tâm

- CSS là gì và cách trình duyệt áp dụng CSS vào HTML.
- Ba cách viết CSS: inline, internal, external.
- Liên kết file CSS bằng thẻ `link`.
- Selector cơ bản: element, class, id.
- Thuộc tính chữ và màu: `color`, `background-color`, `font-size`, `font-family`, `font-weight`, `text-align`, `line-height`.
- Đơn vị cơ bản: `px`, `%`, `em`, `rem`.
- Cách chọn màu và font phù hợp với nội dung.

### Hoạt động của giảng viên

- Tạo file `style.css` và kết nối với project mẫu.
- Giải thích sự khác nhau giữa selector theo thẻ, class và id bằng ví dụ trực tiếp.
- Thiết kế nhanh một phong cách giao diện đơn giản: nền, tiêu đề, đoạn văn, liên kết và nút.
- Nhắc các lỗi thường gặp: sai tên file CSS, đặt nhầm đường dẫn, thiếu dấu chấm ở class.

### Hoạt động của học viên

- Tạo file CSS riêng cho project.
- Gán class cho các khu vực chính trong HTML.
- Định dạng màu nền, màu chữ, font chữ và khoảng cách dòng.
- Làm nổi bật tiêu đề, liên kết và nút gửi form.

### Sản phẩm cần hoàn thành sau buổi học

Website đã có lớp giao diện đầu tiên: màu sắc thống nhất, chữ dễ đọc, các phần chính được phân biệt bằng class và CSS riêng.

### Bài tập về nhà

Thiết kế lại giao diện website với yêu cầu:

- Có bảng màu cơ bản gồm 2-3 màu chính.
- Tiêu đề chính nổi bật nhưng vẫn dễ đọc.
- Ảnh đại diện hoặc ảnh hero được bo góc.
- Danh sách sở thích/kỹ năng được trình bày gọn gàng.
- Không dùng inline CSS trừ khi giảng viên yêu cầu để minh họa.

### Tiêu chí đánh giá ngắn gọn

- CSS được đặt trong file riêng.
- Selector dùng đúng mục đích.
- Màu sắc và chữ đủ tương phản.
- Trang không còn cảm giác là HTML thô.

## Buổi 4: Box model, spacing, border và card layout

### Mục tiêu cụ thể

- Giải thích được vì sao phần tử có kích thước và khoảng cách như đang hiển thị trên trình duyệt.
- Dùng margin, padding, border và border-radius để tạo khối nội dung rõ ràng.
- Xây dựng được card đơn giản cho kỹ năng, dịch vụ, sản phẩm hoặc dự án.

### Kiến thức trọng tâm

- CSS Box Model: content, padding, border, margin.
- Thuộc tính: `width`, `max-width`, `height`, `padding`, `margin`, `border`, `border-radius`, `box-sizing`.
- Display cơ bản: `block`, `inline`, `inline-block`, `none`.
- Cách căn giữa container.
- Tạo card giao diện: ảnh, tiêu đề, mô tả, nút.
- Khoảng trắng trong thiết kế giao diện.

### Hoạt động của giảng viên

- Dùng DevTools để chỉ ra content, padding, border và margin trên một phần tử thật.
- Chuyển một section văn bản thành card có khoảng cách, viền và nền riêng.

## Gợi ý chia điểm chi tiết

### HTML và cấu trúc nội dung - 25 điểm

- Cấu trúc HTML đầy đủ: 5 điểm
- Dùng semantic HTML hợp lý: 5 điểm
- Nội dung rõ ràng, phù hợp với đề tài: 5 điểm
- Link, ảnh và form hiển thị/hoạt động đúng: 5 điểm
- Code HTML sạch, thụt dòng dễ đọc: 5 điểm

### CSS và giao diện - 30 điểm

- Bố cục rõ ràng, dễ theo dõi: 8 điểm
- Màu sắc, font chữ và spacing hợp lý: 6 điểm
- Sử dụng Flexbox/Grid phù hợp: 7 điểm
- Component như button, card, form có thiết kế nhất quán: 5 điểm
- CSS được tổ chức gọn, ít lặp không cần thiết: 4 điểm

### Responsive và trải nghiệm - 15 điểm

- Mobile không bị tràn ngang: 4 điểm
- Layout chuyển đổi hợp lý giữa desktop và mobile: 4 điểm
- Ảnh, card, form và menu dễ sử dụng trên màn hình nhỏ: 4 điểm
- Có hover/focus/transition ở mức vừa phải: 3 điểm

### GitHub và triển khai - 15 điểm

- Repository có cấu trúc đúng: 4 điểm
- Commit rõ ràng, thể hiện quá trình làm: 4 điểm
- README đủ thông tin: 3 điểm
- GitHub Pages deploy thành công: 4 điểm

### Hoàn thiện sản phẩm và trình bày - 15 điểm

- Sản phẩm có nội dung hoàn chỉnh, không còn phần nháp: 5 điểm
- Có cá nhân hóa hoặc ý tưởng riêng: 3 điểm
- Demo mạch lạc trong thời gian quy định: 3 điểm
- Giải thích được cách làm và lựa chọn kỹ thuật: 2 điểm
- Tự đánh giá được điểm mạnh và điểm cần cải thiện: 2 điểm
